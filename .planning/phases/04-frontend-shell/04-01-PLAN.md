    Run from project root:

    1. `npx create-next-app@latest frontend --typescript --tailwind --eslint --app --src-dir --import-alias "@/*"` — accept defaults, do NOT use `--turbopack` flag.

    2. `cd frontend` and verify Tailwind version. If v4 was installed: `npm install tailwindcss@^3.4 tailwindcss-animate@^1` to downgrade (shadcn requires v3). If v3, proceed.

    3. Run `npx shadcn@latest init` — choose: New York style, CSS variables ON, no Tailwind prefix, base color slate. This creates `components.json` and `src/lib/utils.ts` with `cn()`.

    4. Add shadcn components: `npx shadcn@latest add button card sonner`

    5. Install additional deps: `npm install motion next-themes sonner react-textarea-autosize`

    6. Install dev deps for testing: `npm install -D vitest @vitejs/plugin-react jsdom @testing-library/react @testing-library/jest-dom @testing-library/user-event`

    7. Create `frontend/.env.local` with: `NEXT_PUBLIC_API_URL=http://localhost:8000`

    8. Update `tailwind.config.ts` to ensure `darkMode: ['class']` is set (required for next-themes).

    9. Verify `npm run dev` starts without errors, then kill the process.

    IMPORTANT: If `create-next-app` prompts interactively, provide answers matching the flags above. The `--src-dir` flag creates `src/` directory structure.

         - Generic Error -> err.message
         - Unknown -> "An unexpected error occurred."
